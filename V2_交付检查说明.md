# 中国古建筑自由搭建网页 V2 交付检查说明

## 1. 项目背景

这是一个基于现有 V1 工程继续增量开发的 V2 原型项目。

技术栈保持不变：

- Vite
- React
- JavaScript
- Tailwind CSS
- three
- @react-three/fiber
- @react-three/drei

本次开发没有重建脚手架，也没有推翻 V1 的整体界面结构，而是在原项目上继续增强“多对象编辑器”能力。

---

## 2. V2 开发目标

V2 的核心目标是把项目从：

“单体占位展示 + 基础面板交互”

升级为：

“可新增多个构件、可选择对象、可编辑对象、可按图纸辅助搭建、并支持本地保存/恢复的轻量编辑器”

---

## 3. 已完成的 V2 功能

### 3.1 多对象构件放置

已实现点击构件面板按钮后，向场景中新增对象，而不再只是切换单体展示状态。

当前可新增对象包括：

- 台基
- 立柱
- 横梁
- 屋顶
- 飞檐
- 石阶

每个新增对象都会写入 `sceneObjects` 数组，包含以下结构：

```js
{
  id,
  type,
  position: [x, y, z],
  rotation: [x, y, z],
  scale: [x, y, z],
  visible,
  meta: {}
}
```

这些对象会真实渲染到 React Three Fiber 场景中。

---

### 3.2 单对象选择与高亮

已实现：

- 点击场景对象可选中
- 当前对象使用高亮材质表现
- 点击空白区域取消选中
- 属性面板与对象列表同步显示当前选中对象

---

### 3.3 属性编辑升级

属性面板已从编辑“单个总建筑”改为编辑“当前选中对象”。

可编辑内容包括：

- X 方向位置
- Z 方向位置
- Y 高度
- 旋转
- 缩放

可执行操作包括：

- 复制对象
- 隐藏 / 显示对象
- 删除对象

复制对象时会自动偏移位置，避免完全重叠。

---

### 3.4 对象列表 / 图层列表

已新增对象列表区域，支持：

- 显示场景中的所有对象
- 显示对象类型
- 显示对象 id 简写
- 显示当前可见 / 隐藏状态
- 点击列表项选中对象
- 列表内直接显示 / 隐藏对象
- 列表内直接删除对象
- 按类型分组显示

---

### 3.5 轴网与吸附

已实现：

- 更明显的地面网格
- 场景中轴线辅助
- 新增对象默认按网格放置
- 位置 slider 调整时按固定步长吸附
- 避免出现大量小数噪音

当前主要吸附步长为：

- 位置：0.5
- 旋转：15 度
- 缩放：0.1

---

### 3.6 图纸叠加辅助

已实现图纸面板中的“图纸叠加开关”。

打开后，场景中会显示：

- 中轴线
- 基础参考框
- 推荐建筑平面轮廓
- 半透明参考平面

图纸面板中还保留了原有 5 条步骤文案，并新增了推荐布局说明：

- 中央 1 个台基
- 两侧对称立柱
- 上方横梁
- 顶部屋面
- 前侧石阶

---

### 3.7 本地保存 / 读取 / 重置

已实现顶部操作区中的：

- 保存方案
- 读取方案
- 重置方案

保存内容包括：

- `sceneObjects`
- 当前天气
- 当前光影
- 当前视角
- 图纸叠加状态
- 当前选中对象
- 当前步骤卡片显示状态

本地存储 key：

```txt
china-architecture-builder-v2
```

页面首次加载时，如果检测到本地缓存，会自动恢复上次方案。

---

### 3.8 视角体验升级

底部视角按钮已保留并增强：

- 俯视
- 透视
- 图纸叠加
- 自由漫游

当前实现状态：

- `俯视`：真实切换相机至顶部视角
- `透视`：真实切回默认透视角
- `图纸叠加`：真实切换场景蓝图参考层
- `自由漫游`：切换到更适合自由观察的 OrbitControls 预设视角

## 4. 哪些是完整版，哪些是简化版

### 已接近完整版的部分

- 多对象新增与渲染
- 单对象选择
- 属性编辑
- 对象复制 / 删除 / 显隐
- 对象列表管理
- 本地保存 / 读取 / 重置
- 视角按钮交互
- 图纸参考层开关

### 当前属于简化版的部分

- 所有构件仍是几何体占位，不是真实古建筑模型
- 图纸叠加不是 CAD 导入，而是参考线框与占位平面
- 自由漫游仍基于 OrbitControls，没有拖拽 gizmo
- 对象移动目前主要通过属性 slider，而不是场景内直接拖拽

## 8. 建议 ChatGPT 重点检查的内容

请重点检查以下方面：

### 8.1 架构与状态组织

- `App.jsx` 是否承担了过多业务逻辑
- `sceneObjects` 的更新方式是否足够清晰
- 是否还可以进一步拆出 `useSceneObjects` 之类的逻辑层

### 8.2 场景对象设计

- `SceneObjectRenderer.jsx` 是否适合继续扩展更多构件类型
- `sceneObjectFactory.js` 的默认位置与偏移策略是否合理

### 8.3 交互完整性

- 对象选择、取消选择、删除后的状态处理是否严谨
- 本地保存 / 恢复是否还存在边界情况
- 视角切换和图纸叠加逻辑是否足够清晰

### 8.4 UI / UX

- 左侧面板是否仍然足够轻量
- 属性面板是否信息过密或还可继续优化
- 对象列表的可读性和可扩展性是否合理

### 8.5 后续演进建议

请重点给出 V3 最值得优先做的方向，例如：

- 场景内拖拽移动
- 构件吸附规则升级
